fluid.grid

Constrain a 2D DataSet into a Grid.

Description

Maps a set of 2D points in a fluid.dataset to a rectangular grid.

Discussion

fluid.grid transforms a two-dimensional dataset into a grid using a variant of the Jonker-Volgenant algorithm (https://www.gwern.net/docs/statistics/decision/1987-jonker.pdf). This can be useful to generate compact grid layouts from the output of dimensionality reduction algorithms such as fluid.umap, fluid.pca or fluid.mds and for making uniformly distributed spaces out of any two-dimensional dataset.

This approach is similar to projects like CloudToGrid ( https://github.com/kylemcdonald/CloudToGrid/ ), RasterFairy ( https://github.com/Quasimondo/RasterFairy ) or IsoMatch ( https://github.com/ohadf/isomatch ).

Read more about fluid.grid on the learn platform.

fluid.grid is part of the Fluid Decomposition Toolkit of the FluCoMa project. For more explanations, learning material, and discussions on its musicianly uses, visit flucoma.org.

Arguments

Attributes

axis [int] (default: 0)

The axis to which the extent constraint is applied. The default 0 is horizontal, and 1 is vertical.

extent [number] (default: 0)

The size to which the selected axis will be constrained. The default is 0, which turns the constraints off.

Constraints

  • Minimum: 0

oversample [number] (default: 1)

A factor to oversample the destination grid. The default is 1, so the grid has the same number of points as the input. Factors of 2 or more will allow a larger destination grid, which will respect the original shape a little more, but will also be sparser.

Constraints

  • Minimum: 1

warnings [number] (default: 0)

Enable warnings to be issued whenever a parameter value is constrained (e.g. clipped)

Messages
