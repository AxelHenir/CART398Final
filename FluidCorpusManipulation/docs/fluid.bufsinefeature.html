fluid.bufsinefeature

Buffer-Based Sinusoidal Peak Tracking

Description

Interpolated Sinusoidal Peak Tracking on the Spectrum of Audio Stored in a Buffer.

Discussion

This process is tracking peaks in the spectrum of audio stored in a buffer, then estimating an interpolated frequency and amplitude of that peak in relation to its spectral context. It is the first part of the process used by fluid.bufsines.

The process will return two buffers containing time series that describes the interpolated frequencies and magnitudes changing over time in the source buffer.

Read more about fluid.bufsinefeature on the learn platform.

fluid.bufsinefeature is part of the Fluid Decomposition Toolkit of the FluCoMa project. For more explanations, learning material, and discussions on its musicianly uses, visit flucoma.org.

Arguments

Number of Sinusoidal Peaks [number]

Optional
(default: 10)

Shorthand argument for numpeaks

Attributes

detectionthreshold [number] (default: -96.0)

The threshold in dB above which a magnitude peak is considered to be a sinusoidal component.

Constraints

  • Minimum: -144
  • Maximum: 0

fftsettings [int] (default: [['winSize', 1024], ['hopSize', -1], ['fftSize', -1]])

FFT settings consist of three numbers representing the window size, hop size and FFT size in samples:

The hop size and fft size can both be set to -1 (and are by default), with slightly different meanings:

  • For the hop size, -1 = windowsize/2
  • For the FFT size, -1 = windowsize snapped to the nearest equal / greater power of 2 (e.g. windowsize 1024 => fftsize 1024, but windowssize 1000 also => fftsize 1024)

Constraints

  • FFTSize, if != -1, will snap to the next greatest power of two &gt; 4
  • The maximum FFT size is limited to the value of the maxfftsize argument
  • if FFT size != -1, then window size is clipped at FFT size

frequnit [int] (default: 0)

The units and scale used to report the frequency of the peaks. By default (0), it is in Hz (linear), and the alternative (1) is in MIDI (logarithmic).

frequency [symbol] (default: None)

The buffer where the interpolated frequency of the peaks will be written.

magunit [int] (default: 0)

The units and scale used to report the magnitude of the peaks. By default (0), it is in amp (linear), and the alternative (1) is in dB (logarithmic).

magnitude [symbol] (default: None)

The buffer where the interpolated magnitude of the peaks will be written.

maxfftsize [number] (default: -1)

Set an explicit upper bound on the FFT size at object instantiation. The default of -1 sets this to whatever the initial FFT size is

maxnumpeaks [number] (default: -1)

Manually sets a maximum value for numpeaks. Can only be set at object instantiation. The default of -1 sets this equal to the initial value of numpeaks

numchans [number] (default: -1)

For multichannel sources, the number of channels to analyse.

numframes [number] (default: -1)

The duration (in samples) to analyse.

numpeaks [number] (default: 10)

The number of peaks to search report back. It is capped at (fftSize / 2) + 1.

Constraints

  • Minimum: 1
  • Maximum: MIN((fft size / 2) + 1 (see fft settings), maxnumpeaks)

order [int] (default: 0)

How the reported peaks are to be ordered. By default (0), it is by frequencies (lowest first), and the alternative (1) is by magnitude (loudest first).

padding [int] (default: 1)

Controls the zero-padding added to either end of the source buffer or segment. Padding ensures all values are analysed. Possible values are:

  1. No padding - The first analysis window starts at time 0, and the samples at either end will be tapered by the STFT windowing function.

  2. Half the window size - The first sample is centred in the analysis window ensuring that the start and end of the segment are accounted for in the analysis.

  3. Window size minus the hop size - Mode 2 can be useful when the overlap factor (window size / hop size) is greater than 2, to ensure that the input samples at either end of the segment are covered by the same number of analysis frames as the rest of the analysed material.

source [symbol] (default: None)

The array to use as the source material. The channels of multichannel buffers will be processed sequentially.

startchan [number] (default: 0)

For multichannel sources, the starting channel to analyse.

Constraints

  • Minimum: 0

startframe [number] (default: 0)

The starting point for analysis in the source (in samples).

Constraints

  • Minimum: 0

warnings [number] (default: 0)

Enable warnings to be issued whenever a parameter value is constrained (e.g. clipped)

Messages

bang

A bang will commence processing on object given its current settings, unless it is already busy. In that case, the behaviour depends on the value of the queue attribute.

cancel

When processing in non-blocking mode, this will cancel the job underway, without updating destination buffers.

reset

Calling reset will change parameter values back to those the object was created with, i.e. any attribute values set in the box will be retained
