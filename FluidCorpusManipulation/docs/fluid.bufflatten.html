fluid.bufflatten

Flatten a multichannel buffer

Description

Flatten a multichannel array to a single channel. This can be useful to structure a buffer such that it can be added to a fluid.dataset

Discussion

The axis argument determines how the flattening is arranged. The default value, 1, flattens channel-wise (similar to how audio files are stored), such that (if we imagine channels are rows, time positions are columns):

a 1 a 2 a 3
b 1 b 2 b 3
c 1 c 2 c 3

becomes

a 1 b 1 c 1 a 2 b 2 c 2 a 3 b 3 c 3

whereas with axis = 0, the buffer is flattened frame-wise, resulting in:

a 1 a 2 a 3 b 1 b 2 b 3 c 1 c 2 c 3

To learn more visit https://learn.flucoma.org/reference/bufflatten/.

Read more about fluid.bufflatten on the learn platform.

fluid.bufflatten is part of the Fluid Decomposition Toolkit of the FluCoMa project. For more explanations, learning material, and discussions on its musicianly uses, visit flucoma.org.

Arguments

Attributes

axis [int] (default: 1)

Whether to flatten the buffer channel-wise (1) or frame-wise (0). The default is 1 (channel-wise).

destination [symbol] (default: None)

The array to write the flattened data to.

numchans [number] (default: -1)

For multichannel source buffers, how many channels should be processed. The default of -1 indicates to process up through the last channel in source.

numframes [number] (default: -1)

How many frames should be processed. The default of -1 indicates to process through the end of the buffer.

source [symbol] (default: None)

The array to flatten.

startchan [number] (default: 0)

For multichannel source buffers, which channel to begin the processing. The default is 0.

Constraints

  • Minimum: 0

startframe [number] (default: 0)

Where in source should the flattening process start, in samples. The default is 0.

Constraints

  • Minimum: 0

warnings [number] (default: 0)

Enable warnings to be issued whenever a parameter value is constrained (e.g. clipped)

Messages

bang

A bang will commence processing on object given its current settings, unless it is already busy. In that case, the behaviour depends on the value of the queue attribute.

cancel

When processing in non-blocking mode, this will cancel the job underway, without updating destination buffers.

reset

Calling reset will change parameter values back to those the object was created with, i.e. any attribute values set in the box will be retained
