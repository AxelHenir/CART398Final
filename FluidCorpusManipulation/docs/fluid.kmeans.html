fluid.kmeans

Cluster data points with K-Means

Description

Uses the K-means algorithm to learn clusters from a fluid.dataset.

Discussion

kmeans facilitates learning of clusters from a fluid.dataset. This allows you to assign each point in the data a discrete membership to a group or cluster. The algorithm works by partitioning points into discrete clumps that ideally have equal variance. See the Scitkit-learn reference for a more technical explanation: https://scikit-learn.org/stable/modules/clustering.html#k-means

Read more about fluid.kmeans on the learn platform.

fluid.kmeans is part of the Fluid Decomposition Toolkit of the FluCoMa project. For more explanations, learning material, and discussions on its musicianly uses, visit flucoma.org.

Arguments

Attributes

maxiter [number] (default: 100)

The maximum number of iterations the algorithm will use whilst fitting.

Constraints

  • Minimum: 1

numclusters [number] (default: 4)

The number of clusters to classify data into.

Constraints

  • Minimum: 1

warnings [number] (default: 0)

Enable warnings to be issued whenever a parameter value is constrained (e.g. clipped)

Messages

fit

Arguments: dataSet (Unknown Type (input dataset))

Identify numclusters clusters in a fluid.dataset. It will optimise until no improvement is possible, or up to maxiter, whichever comes first. Subsequent calls will continue training from the stopping point with the same conditions.

Argument details:
  • dataSet:

    A fluid.dataset of data points.

predict

Arguments: dataSet (Unknown Type (input dataset)), labelSet (symbol)

Given a trained object, return the cluster ID for each data point in a fluid.dataset to a fluid.labelset.

Argument details:
  • dataSet:

    A fluid.dataset containing the data to predict.

  • labelSet:

    A fluid.labelset to retrieve the predicted clusters.

transform

Arguments: srcDataSet (Unknown Type (input dataset)), dstDataSet (symbol)

Given a trained object, return for each item of a provided fluid.dataset its distance to each cluster as an array, often referred to as the cluster-distance space.

Argument details:
  • srcDataSet:

    A fluid.dataset containing the data to transform.

  • dstDataSet:

    A fluid.dataset to contain the new cluster-distance space.

predictpoint

Arguments: buffer (symbol)

Given a trained object, return the cluster ID for a data point in a array

Argument details:
  • buffer:

    A array containing a data point.

transformpoint

Arguments: sourceBuffer (symbol), targetBuffer (symbol)

Given a trained object, return the distance of the provided point to each cluster centroid. Both points are handled as array

Argument details:
  • sourceBuffer:

    A array containing a data point.

  • targetBuffer:

    A array to write in the distance to all the cluster centroids.

fittransform

Arguments: srcDataSet (Unknown Type (input dataset)), dstDataSet (symbol)

Run fit and transform in a single pass: i.e. train the model on the incoming fluid.dataset and then return its cluster-distance space in the destination fluid.dataset

Argument details:
  • srcDataSet:

    A fluid.dataset containing the data to fit and transform.

  • dstDataSet:

    A fluid.dataset to contain the new cluster-distance space.

getmeans

Arguments: dataSet (symbol)

Given a trained object, retrieve the means (centroids) of each cluster as a fluid.dataset

Argument details:
  • dataSet:

    A fluid.dataset of clusters with a mean per column.

setmeans

Arguments: dataSet (Unknown Type (input dataset))

Overwrites the means (centroids) of each cluster, and declare the object trained.

Argument details:
  • dataSet:

    A fluid.dataset of clusters with a mean per column.

fitpredict

Arguments: dataSet (Unknown Type (input dataset)), labelSet (symbol)

Run fit and predict in a single pass: i.e. train the model on the incoming fluid.dataset and then return the learned clustering to the passed fluid.labelset

Argument details:
  • dataSet:

    A fluid.dataset containing the data to fit and predict.

  • labelSet:

    A fluid.labelset to retrieve the predicted clusters.
