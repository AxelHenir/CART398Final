fluid.bufnoveltyfeature

Buffer-Based Novelty Feature

Description

Calculates the novelty feature of audio stored in a buffer.

Discussion

Calculate novelty of audio stored in a buffer, the feature used by fluid.bufnoveltyslice to perform segmentation.

Novelty is derived by running a kernel across the diagonal of the similarity matrix. It implements the seminal results published in 'Automatic Audio Segmentation Using a Measure of Audio Novelty' by J Foote.

The process will return a buffer containing a time series that describes the novelty feature changing over time in the source buffer.

Read more about fluid.bufnoveltyfeature on the learn platform.

fluid.bufnoveltyfeature is part of the Fluid Decomposition Toolkit of the FluCoMa project. For more explanations, learning material, and discussions on its musicianly uses, visit flucoma.org.

Arguments

KernelSize [number]

Optional
(default: 3)

Shorthand argument for kernelsize

Smoothing Filter Size [number]

Optional
(default: 1)

Shorthand argument for filtersize

Attributes

algorithm [int] (default: 0)

The feature on which novelty is computed.

  1. Spectrum – The magnitude of the full spectrum.

  2. MFCC – 13 Mel-Frequency Cepstrum Coefficients.

  3. Chroma – The contour of a 12-band chromagram.

  4. Pitch – The pitch and its confidence.

  5. Loudness – The true peak and loudness.

features [symbol] (default: None)

The buffer where the novelty feature will be written.

fftsettings [int] (default: [['winSize', 1024], ['hopSize', -1], ['fftSize', -1]])

FFT settings consist of three numbers representing the window size, hop size and FFT size in samples:

The hop size and fft size can both be set to -1 (and are by default), with slightly different meanings:

  • For the hop size, -1 = windowsize/2
  • For the FFT size, -1 = windowsize snapped to the nearest equal / greater power of 2 (e.g. windowsize 1024 => fftsize 1024, but windowssize 1000 also => fftsize 1024)

Constraints

  • FFTSize, if != -1, will snap to the next greatest power of two &gt; 4
  • The maximum FFT size is limited to the value of the maxfftsize argument
  • if FFT size != -1, then window size is clipped at FFT size

filtersize [number] (default: 1)

The size of a smoothing filter that is applied on the novelty curve. A larger filter size allows for cleaner cuts on very sharp changes.

Constraints

  • Minimum: 1
  • Maximum: maxfiltersize

kernelsize [number] (default: 3)

The granularity of the window in which the algorithm looks for change, in samples. A small number will be sensitive to short term changes, and a large number should look for long term changes.

Constraints

  • Minimum: 3
  • Maximum: maxkernelsize
  • Snaps to odd numbers

maxfftsize [number] (default: -1)

Set an explicit upper bound on the FFT size at object instantiation. The default of -1 sets this to whatever the initial FFT size is

maxfiltersize [number] (default: -1)

Manually sets a maximum value for filtersize. Can only be set at object instantiation. The default of -1 sets this equal to the initial value of filtersize

maxkernelsize [number] (default: -1)

Manually sets a maximum value for kernelsize. Can only be set at object instantiation. The default of -1 sets this equal to the initial value of kernelsize

numchans [number] (default: -1)

For multichannel srcBuf, how many channels should be summed.

numframes [number] (default: -1)

How many frames should be processed.

padding [int] (default: 1)

Controls the zero-padding added to either end of the source buffer or segment. Padding ensures all values are analysed. Possible values are:

  1. No padding - The first analysis window starts at time 0, and the samples at either end will be tapered by the STFT windowing function.

  2. Half the window size - The first sample is centred in the analysis window ensuring that the start and end of the segment are accounted for in the analysis.

  3. Window size minus the hop size - Mode 2 can be useful when the overlap factor (window size / hop size) is greater than 2, to ensure that the input samples at either end of the segment are covered by the same number of analysis frames as the rest of the analysed material.

source [symbol] (default: None)

The buffer to use as the source material to be sliced through novelty identification. The different channels of multichannel buffers will be summed.

startchan [number] (default: 0)

For multichannel srcBuf, which channel should be processed.

Constraints

  • Minimum: 0

startframe [number] (default: 0)

Where in the srcBuf should the slicing process start, in samples.

Constraints

  • Minimum: 0

warnings [number] (default: 0)

Enable warnings to be issued whenever a parameter value is constrained (e.g. clipped)
