fluid.bufthreaddemo

A Tutorial Object to Experiment with Multithreading in FluidBuf* Objects

Description

Implements a tutorial object to illustrate and experiment with the behaviour of the FluidBuf* objects. To simulate their behaviour in various blocking modes, the object after waiting for time milliseconds, return its delay length as a float writen at index [0] of the specified destination buffer.

Discussion

Read more about fluid.bufthreaddemo on the learn platform.

fluid.bufthreaddemo is part of the Fluid Decomposition Toolkit of the FluCoMa project. For more explanations, learning material, and discussions on its musicianly uses, visit flucoma.org.

Arguments

Attributes

result [symbol] (default: None)

The destination buffer, where the value should be written at the end of the process.

time [number] (default: 0.0)

The duration in milliseconds of the delay that the background thread will wait for before yielding the value to the destination buffer.

Constraints

  • Minimum: 0.0

warnings [number] (default: 0)

Enable warnings to be issued whenever a parameter value is constrained (e.g. clipped)

Messages

bang

A bang will commence processing on object given its current settings, unless it is already busy. In that case, the behaviour depends on the value of the queue attribute.

cancel

When processing in non-blocking mode, this will cancel the job underway, without updating destination buffers.

reset

Calling reset will change parameter values back to those the object was created with, i.e. any attribute values set in the box will be retained
