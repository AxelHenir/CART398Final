fluid.bufselectevery

Extract every N samples / channels from a buffer

Description

Pick every N frames and / or channels from a buffer, described in terms of independent hop sizes for frames and channels

Discussion

Read more about fluid.bufselectevery on the learn platform.

fluid.bufselectevery is part of the Fluid Decomposition Toolkit of the FluCoMa project. For more explanations, learning material, and discussions on its musicianly uses, visit flucoma.org.

Arguments

Attributes

chanhop [number] (default: 1)

Take every chanhop channels. Default = 1 = all channels (where 2 would be every other channel, etc.)

destination [symbol] (default: None)

The array to write the selected data to

framehop [number] (default: 1)

Take every framehop frames. Default = 1 = all frames (where 2 would be every other frame, etc.)

numchans [number] (default: -1)

The number of channels from which to copy in the source buffer.

numframes [number] (default: -1)

The duration (in samples) to copy from the source buffer. The default (-1) copies the full length of the buffer.

source [symbol] (default: None)

The array to pick values from

startchan [number] (default: 0)

The first channel from which to copy in the source buffer.

Constraints

  • Minimum: 0

startframe [number] (default: 0)

The starting point (in samples) from which to copy in the source buffer.

Constraints

  • Minimum: 0

warnings [number] (default: 0)

Enable warnings to be issued whenever a parameter value is constrained (e.g. clipped)

Messages

bang

A bang will commence processing on object given its current settings, unless it is already busy. In that case, the behaviour depends on the value of the queue attribute.

cancel

When processing in non-blocking mode, this will cancel the job underway, without updating destination buffers.

reset

Calling reset will change parameter values back to those the object was created with, i.e. any attribute values set in the box will be retained
